Research in progress

Adversarial Classification of ML Training on GPUs

Can always-available GPU telemetry detect ML training — even when someone is trying to hide it?

A SPAR research project · GitHub

Motivation

Hardware governance proposals increasingly suggest monitoring GPUs to detect unauthorized frontier model training. But how reliably can we distinguish ML training from other intensive workloads like scientific computing, rendering, or crypto mining — using only the telemetry that cloud providers already collect? And what happens when the user actively tries to disguise their workload?

Research questions

  • Which GPU telemetry signals most clearly distinguish ML training from other workloads?
  • Can low-overhead Tier 1 metrics (nvidia-smi level) substitute for expensive per-kernel profiling?
  • What temporal patterns — epoch periodicity, memory growth, forward/backward asymmetry — are unique to training?
  • Can a determined adversary disguise training to evade detection, and at what cost?
  • What should a hardware-embedded monitoring module measure to catch unauthorized training?

Approach

We collect GPU telemetry across a three-tier monitoring hierarchy while running a diverse set of GPU workloads on data-center GPUs (A100, H100):

TierSourceMetricsOverhead
1NVML (pynvml)GPU/mem utilization, power, temperature, clocks, PCIe throughput~0%
2DCGM profilingTensor core activity, FP16/32/64 pipe utilization, SM occupancy, DRAM bandwidth~0%
3Nsight ComputePer-kernel PMCs: instruction mix, memory transactions, FLOP counts1,200–5,300%

We then build a temporal-pattern classifier that uses features like autocorrelation (epoch periodicity), memory trajectory slope, rolling utilization variance, and compute duty cycle to classify workloads. This temporal approach is novel — prior work (WAVE, Differential Architecture) uses only aggregate or per-kernel statistics.

Related work

WAVE (Xu et al., ASPLOS 2026) fingerprints LLM inference workloads using Nsight Compute PMCs with 87–93% accuracy, but at 1,200–5,300% overhead. Differential Architecture (Anonymous, ISCA 2026) analytically characterizes hardware bottlenecks across GPU workload classes. Our work extends both by focusing on ML training (not just inference), using zero-overhead Tier 1 telemetry, exploiting temporal patterns, and evaluating adversarial evasion.
